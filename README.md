# Webカメラのフェイストラッキングによる立体視デモシステム

## 概要
Webカメラを用いてユーザーの顔をトラッキングし、モニターの奥に空間があるように錯覚させる立体視（Portalgraph風）システムです。
Python側で取得した顔の座標データをUDP通信でUnityへ送信し、リアルタイムでカメラワークに反映させることで、没入感の高い疑似3D体験を実現しました。

## 動画[https://youtu.be/snJV44eKH0w]

## システム構成 (Architecture)
- **顔認識・データ送信**: Python (OpenCV)
- **3D環境・描画**: Unity (C#)
- **通信プロトコル**: UDPソケット通信 (ローカルネットワーク)

## 主要な機能
- **リアルタイムUDP通信**: PythonとUnityのプロセス間で、顔の座標データを低遅延かつ高速に送受信します。
- **円軌道（極座標系）アルゴリズム**: 単純な平面スライド移動ではなく、顔の移動量を角度に変換し、三角関数を用いて対象の周囲を回り込む自然なカメラ軌道を計算しています。
- **自動注視（LookAt）機能**: カメラが回り込んだ際に対象が見切れるのを防ぐため、注視点（Target）を分離し、常にキャラクターを画面中央に捉え続ける制御を実装しました。
- **調整用パラメータの公開**: 湾曲の強さ、カメラの距離、高さなどをUnityのインスペクターから実行中にリアルタイム調整できる、保守性と拡張性の高い設計です。

## 動作環境
- **OS**: Windows 10 / 11
- **必須デバイス**: Webカメラ
- **Python**: 3.10.x (※お使いのバージョンに合わせて適宜変更してください)
- **Unity**: 2022.3 LTS (※お使いのバージョンに合わせて適宜変更してください)

## 重要なライブラリ
- opencv-python
- numpy

## 環境構築

### Python側のセットアップ
仮想環境を作成し、必要なパッケージをインストールして実行準備を行います。

```bash
python -m venv venv
venv\Scripts\activate
pip install opencv-python numpy
```

### Unity側のセットアップ
1. Unityプロジェクトを開き、シーンに対象となる3Dモデルを配置します。
2. カメラ制御用のC#スクリプト（UDPReceiver）をMain Cameraなどのオブジェクトにアタッチします。
3. インスペクターから注視点（胸の高さに配置した空のオブジェクト等）をセットし、トラッキング用の各種パラメータを調整します。
